## Session 8 - Deterministic Statement Classification and Local Model Matrix

This session added a small but important control layer before any future write
path: a deterministic utterance classifier. Instead of letting every natural
language input drift toward the query path, the repo now has a first explicit
router for:

- query
- hard fact
- tentative fact
- correction
- preference
- session context

The implementation lives in `src/parser/statement_classifier.py` and is exposed
through the MCP server as `classify_statement`. The point was not to pretend the
full pre-thinker exists yet. The point was to create an honest, deterministic
bridge that already helps small local models behave better.

That bridge immediately proved useful. We exercised the tool through LM Studio's
local `api/v1/chat` endpoint with attached MCP integration and observed the
model-visible reasoning summaries plus tool calls directly. This mattered
because it let us separate three different kinds of failure:

- the model did not call the tool
- the model called the tool but misunderstood the output
- the model understood the output but still answered outside the task boundary

We then ran a small local model matrix across:

- `nvidia/nemotron-3-nano-4b`
- `qwen3.5-4b`
- `qwen/qwen3.5-9b`
- `qwen3.5-27b@q4_k_m`
- `qwen3.5-27b@q6_k`

using the same five classifier prompts:

- `My mother was Ann.`
- `Actually, I meant Alice.`
- `Maybe my mother was Ann.`
- `Keep responses concise.`
- `Who is John's parent?`

All tested models called `classify_statement` correctly and received the
expected classification kinds. The biggest behavioral difference was not
classification accuracy but response discipline. The Nemotron 4B model was the
most likely to blur `report the classification` with `act on the instruction`,
especially on the preference case. The Qwen family, even at 4B, was better at
staying inside the task.

The largest practical takeaway was that `qwen3.5-27b@q4_k_m` looks like a very
strong local evaluator for this project. It preserved the classification quality
of the larger model family while being materially faster than `qwen3.5-27b@q6_k`
on the same MCP workflow.

The evaluation write-up now lives in `docs/research/lmstudio-classifier-matrix.md`.
